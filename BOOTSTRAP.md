## STEP 3: Install Hooks

Three Claude Code hooks handle session continuity at the shell level,
independent of model behaviour:

- **`SessionStart`** — starts the service if needed, injects open goals
  and last snapshot as `additionalContext` so every session begins with
  full operational awareness automatically.
- **`PreCompact`** — writes a state snapshot to memory-graph before
  context compaction so post-compaction sessions can recover.
- **`SessionEnd`** — writes a final snapshot on controlled shutdown.

Make the hook scripts executable:

```bash
chmod +x ~/code/memory-graph/agent/hooks/session-start.sh
chmod +x ~/code/memory-graph/agent/hooks/pre-compact.sh
chmod +x ~/code/memory-graph/agent/hooks/session-end.sh
```

Register them in `~/.claude/settings.json`. If the file does not exist,
create it; if it already has a `hooks` key, merge carefully:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/code/memory-graph/agent/hooks/session-start.sh"
          }
        ]
      }
    ],
    "PreCompact": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/code/memory-graph/agent/hooks/pre-compact.sh"
          }
        ]
      }
    ],
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/code/memory-graph/agent/hooks/session-end.sh"
          }
        ]
      }
    ]
  }
}
```

Verify the hooks are registered:

```bash
cat ~/.claude/settings.json | jq '.hooks | keys'
# Expected: ["PreCompact", "SessionEnd", "SessionStart"]
```

## STEP 4: Write Cron Prompt Files

Write each file below to `~/.claude/prompts/`. These files are the
canonical cron specifications; the heartbeat cron recreates any missing
cron jobs from them on each run.

### cron-02-watchdog.md

```
You are running as a scheduled watchdog (every 6h at :23).

Check all active cron jobs. For any cron with a 7-day TTL that will
expire within the next 24 hours, alert the user via Discord with the
job name and time remaining. Recreate the expiring job immediately
using CronCreate with the content from ~/.claude/prompts/.

Report: "Watchdog: N crons checked, M expiring soon." If M > 0, list
each one.
```

### cron-04-heartbeat.md

```
You are running as the system heartbeat (every 1h at :43).

1. List all active cron jobs. For each prompt file in
   ~/.claude/prompts/, confirm the corresponding cron is running.
   Recreate any that are missing using CronCreate.
2. Check memory API health: curl -s http://127.0.0.1:7777/health.
   If down, start the service and notify the user.
3. Once per calendar day (check /kv/heartbeat_social_sent_date before
   acting), send one social message to the user via Discord — a brief
   observation, question, or update. Do not send between 00:00–08:00.
4. If any alerts were raised in steps 1 or 2, skip the social message.

If POST /cron/active returns 200, sync is complete. If it returns 404, skip — the cron snapshot surface is not yet implemented.
```


### cron-14-predictions-resolver.md

```
You are running as the predictions resolver (daily at 21:53).

Note: this cron requires /wm/prediction which is a planned surface. Check availability: if `curl -sf http://127.0.0.1:7777/wm/prediction/list` returns 404, exit without writing anything.

1. Fetch unresolved predictions past their due date:
   GET /wm/prediction/list?resolved=false
   Filter for due_at < now.
2. For each overdue prediction where the outcome is now clear from
   recent context or conversation history, resolve it:
   PATCH /wm/prediction/<id>/resolve with {actual_outcome, correct}.
3. For predictions where the outcome is still ambiguous, leave them
   unresolved and note them in a daily summary memory.

This builds calibration signal over time. Do not force resolutions
when evidence is genuinely unclear.
```
